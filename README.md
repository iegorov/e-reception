# Electronic reception

В данном приложении я хочу продемонстрировать свое видение фиче-ориентированной архитектуры приложения. По сути - бизнесу важны функциональные возможности приложения (фичи), задачи приходят на реализацию определенной фичи и мы работаем постоянно в рамках какой-то фичи. Мне думается сгуппировать файлы приложения по фичам - неплохая идея. Думаю лично мне было бы удобно искать определенное место в коде, когда я знаю к какой фиче эта логика относится.
Важное замечание, в этом приложение разбиение на фичи выполено довольно крупными кусками. Данное приложение не содержит много логики и такое разбиение не усложняет понимание и не уменьшает степень контроля за приложением, но возможно в крупных проектах разбиение должно быть выполнено меньшими кусками.
